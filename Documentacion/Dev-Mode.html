El modo Developer lo único que hace, (por ahora), es registrar el $response que devuelve la función sapidk_mp internamente en un archivo .txt llamado sapidkmp_log.

Esta función, extremadamente útil para cuando se trata de un desarrollo, nos permite guardar la respuesta que nos realiza la función que ejecutemos y así de este modo tener registro de cual fue el error/respuesta que obtuvimos de la consulta que realizamos.

¿Como lo hago?

Simplemente hay que adicionar este fragmento de código:


"developer" => array(  

    'log' => array( 

        "registro" => true, 

        "directorio" => "" 

    ), 

),

Simplemente indicando como True a registro ya empezara a registrar el log. Ahora, este se guardará en la raíz donde se halla iniciado la función sapidk_mp si hay más de una consulta a la función en distintos directorios, en cada uno de ellos guardara un sapidkmp_log.txt con el registro de $response.

Esto puede cambiar indicando en directorio la raíz "hacia atrás" donde se desea guardar.

Quiero decir, que si el directorio donde se encuentra es "a/b/c" y nosotros no le indicamos donde guardar el log, lo guardara en la carpeta "c", ahora si le indicamos en directorio que lo guarde en "../" lo guardara en la carpeta "b".


"developer" => array(  

    'log' => array( 

        "registro" => true, 

        "directorio" => "../" 

    ), 

),
