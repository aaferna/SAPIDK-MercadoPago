Bien, acá dependemos de cada proyecto, pero detallare las opciones que tenemos actualmente.

Todo se centra en una única función, cual envía un Array, es necesario que las primeras dos lineas de este sean accessToken y user_id

Posterior de este, tenemos tres operadores get post y delete donde pueden contener un array o determinantes de true / false

Un ejemplo es el de la consulta de un usuario, en el ejemplo siguiente consultaremos nuestro usuario para ver que datos nos devuelve, todos los datos que devuelve SAPIDK están preparados para ser trabajados por un Array.

La forma de empezar a trabajar es de la siguiente manera:

$dataArray = array(

    "accessToken" => "APP_USR-1864213086135942-012340-8b0cb3af04c2567f7bb8d09779aefba5-264334950",
    "user_id" => "264334950",
        "get" => array( 
            'user' => true,
            'id' => "264334950" 
        ),

);

print_r(sapidk_mp($dataArray));

Si nos fijamos, el id que se consulta, es nuestro user_id, de esta forma obtendremos nuestra información, como así también la de cualquier otro usuario que trabajemos

La única función a utilizar es sapidk_mp($dataArray); según como tengamos armado nuestro array será la respuesta que reciba el mismo ya que en la estructura del mismo declararemos la acción a realizar.

Recomiendo utilizar $response = sapidk_mp($dataArray); para tener bien ordenado el código, luego de ahí elegir si hacer un print_r($response); o si trabajar la respuesta de otra manera.

En cada ejemplo que está documentado desde aquí, vera directamente el array a utilizar en la función.
