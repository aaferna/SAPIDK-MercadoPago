Según corresponda, se puede hacer una paginación de resultados con los tag limit y offset en la consulta de get, algunas respuestas desde Mercado Pago viene con un limite bajo de resultados que podemos modificar desde estas variables.

En limit y offset, puede ir de 0 a el valor que quieran agregar, siendo 0 o "" el que le dará lugar al valor el defecto que pone Mercado Pago.

Un ejemplo de esto seria así:

$dataArray = array(
"accessToken" => "APP_USR-1864213086135942-012340-8b0cb3af04c2567f7bb8d09779aefba5-264334950",
"user_id" => "264334950",
    "developer" => array( 
        'log' => array(
            "registro" => true,
            "directorio" => ""
        ),
    ),

    "get" => array(
        'qr' => array(
            "list" => true,
            "limit" => "10",
            "offset" => "0"
        ),
    ),

);

¿Como lo uso?

limit

Delimita la cantidad de resultados que disponemos en la consulta, para esto nos sirve que tan lento nos puede realizar la consulta, por ejemplo.

offset

Nos permite mover en la "altura" del resultado, si listamos 1200 resultados y queremos el 1199 en offset colocamos 1199 para listar únicamente ese resultado.

El ejemplo sería el siguiente :

    "developer" => array( 
        'log' => array(
            "registro" => true,
            "directorio" => ""
        ),
    ),

    "get" => array(
        'qr' => array(
            "list" => true,
            "limit" => "1200",
            "offset" => "1199"
        ),
    ),
