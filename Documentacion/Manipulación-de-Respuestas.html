En desarrollo

Desde la version 2.4.0, SAPIDK MP devuelve las llamadas bajo un array con el metodo empleado. Si se ejecuto un GET, la respuesta de esta consulta seria la siguiente:

Array
(
    [get] => Array
)

Si se ejecutan funciones por separado como un DELETE y un POST sin utilizar forWhile este devolverá por consulta, no es conveniente utilizar multiples funciones básicas (GET, POST, PUT o DELETE), ya que se podría "Pisar la información", para eso, reitero la utilización de forWhile
